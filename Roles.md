# External Public Documentation

This document describes the structure and behavior of the public-facing documentation generated by the CMS.

## 1. Overview of the Public Portal

The external documentation site is the primary output of the Docusaurus engine. It is designed to be fast, SEO-friendly, and accessible to end-users without requiring authentication.

*   **Static Generation:** The public site is compiled ahead of time into static HTML/CSS/JS. This guarantees rapid page load speeds globally and ensures search engines can easily index the content.
*   **Navigation & Layout:** The public site utilizes the standard Docusaurus navbar, sidebar, and footer. However, the custom logo and site title updated via the Admin panel are dynamically injected here during the build process.
*   **Separation of Concerns:** The public site is completely unaware of the Internal Workspace. No editing tools, publish requests, or administrative data are bundled into the public JavaScript, ensuring maximum security and performance.

## 2. Content Pipeline (How Content Arrives Here)

Content is never edited directly in the public `docs/` folder via the browser. Instead, it follows a strict pipeline:

1.  **Drafting:** Developers write documentation in the Internal Workspace.
2.  **Review:** A "Publish Request" is submitted.
3.  **Approval & Merge:** An Admin approves the request. The backend copies the Markdown files from the hidden `workspaces/` directory into the public `docs/` directory.
4.  **Rebuild:** If necessary, the server triggers a Docusaurus rebuild to re-compile the newly added Markdown files into public static HTML.

## 3. Public Interactive Elements

While the core text is static HTML generated from Markdown, the public site retains React interactivity for the end-user:
*   **Theme Toggling:** Users can switch between dark and light modes seamlessly.
*   **Code Blocks:** Syntax highlighted code blocks with "Copy to Clipboard" functionality.
*   **Search:** Built-in local search (or Algolia integration) indexes the public `docs/` folder, allowing users to find content instantly.
